# Security Policy

## Supported Versions

This project only provide updates for latest releases. We recommend always using the latest version to ensure you have the latest security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately:

- email: [janreynald.pangilinan@gmail.com](mailto:janreynald.pangilinan@gmail.com)
- twitter: [@redpngilinan](https://twitter.com/redpngilinan)

Please include as much detail as possible so we can validate and fix the issue quickly:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and affected components
- Any proof-of-concept, logs, or screenshots

### What to expect

- We aim to acknowledge reports within 72 hours.
- If confirmed, we will work on a fix and coordinate a responsible disclosure timeline.
- Please do not publicly disclose the vulnerability until a fix is available.
